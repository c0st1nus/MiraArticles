### ⚠️ Замечания (не блокируют)

1. **`[parser.ts:25-28]`** — Regex с флагом `/g` имеют состояние (lastIndex). При многократных вызовах `cleanDraftText()` с тем же паттерном возможны баги. Рекомендация: убрать флаг `g` или создавать regex внутри функции.

2. **`[client.ts:37-40]`** — `getBotUsername()` не валидирует пустую строку после `replace(/^@/, "")`. Если `MIRA_BOT_USERNAME="@"`, получим пустой username. Добавить проверку:
   ```ts
   if (!name) throw new Error("Invalid MIRA_BOT_USERNAME");
   ```

3. **`[client.ts:123-137]`** — `considerDone()` использует `return` внутри Promise executor для раннего выхода. Это работает, но читается как possible typo (возможно имелся в виду `finish()`). Не баг, но стоит перепроверить логику.

4. **`[parser.ts:49]`** — fallback для неизвестных типов кнопок возвращает `{ text: "text" ... }`, что выглядит как placeholder. Лучше логировать неизвестный тип для диагностики.
