# Протокол бота @mira (Telegram)

Документ для интеграции MiraArticles. **Заполняется по мере разведки** — отметки `[ ]` / `[x]`.

## 1. Учётные данные (локально)

| Файл | Поля | Когда |
|------|------|--------|
| [`token/telegram.json`](../token/telegram.json) | `api_id`, `api_hash` | Статично, с https://my.telegram.org |
| `token/telegram.session` | StringSession (GramJS) | **После первого** `bun run scripts/telegram-login.ts` (создаётся, в git не класть) |

Сессия **не** хранится в `telegram.json` — только в `.session` / env `TELEGRAM_SESSION`.

## 3. MTProto (техническая схема)

```
Scheduler
  → GramJS client (StringSession from token/telegram.session)
  → messages.SendMessage(peer=@mira, message=prompt)
  → wait NewMessage (correlation: after requestId / timeout 120s)
  → parse text + ReplyInlineMarkup
  → optional: click callback button (state machine)
  → return draftText
```

- Библиотека: [GramJS](https://gram.js.org/) (`telegram` npm).
- Один активный запрос к боту за раз.
- `FLOOD_WAIT`: sleep(seconds) + retry once.

## 6. Ошибки

| Симптом | Действие |
|---------|----------|
| Нет ответа 120s | fail cycle, log, skip publish |
| FloodWaitError | sleep, retry |
| Session revoked | re-run `telegram-login.ts` |
| Bot blocked | alert user |
